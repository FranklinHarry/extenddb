# ExtendDB

ExtendDB is an open source DynamoDB-compatible API adapter that translates DynamoDB API calls into queries against a pluggable storage backend. PostgreSQL is the reference backend. Apache 2.0, maintained by AWS.

## Repo layout

```
crates/
├── auth/              SigV4 signature verification and IAM policy evaluation
├── bin/               CLI binary (init, serve, manage, status, stop, destroy)
├── core/              DynamoDB data model, expression parsing, validation
├── engine/            Request routing, operation dispatch
├── server/            HTTP/TLS server, request decoding, response encoding
├── storage/           Storage trait (pluggable backend interface)
└── storage-postgres/  PostgreSQL backend implementation
tests/                 Integration tests (Python + Rust)
samples/               sample_app.py, stream_consumer.py
scripts/               install-linux.sh, install-macos.sh
docs/                  User-facing documentation
```

## Build

```bash
cargo build --release
```

Binary: `target/release/extenddb`

Minimum toolchain: Rust 1.85, PostgreSQL 14.

## Test

### Rust unit tests

```bash
cargo test
```

### Integration tests (Python)

Requires a running ExtendDB instance and Python venv with boto3:

```bash
source .venv/bin/activate
pytest tests/ -v
```

### Lint

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
```

## Run locally

```bash
./target/release/extenddb init        # creates extenddb.toml, TLS cert, Postgres schemas
./target/release/extenddb serve --config extenddb.toml
./target/release/extenddb status --config extenddb.toml
```

Default port: 8000 (configurable in `extenddb.toml` under `[server]`).

## CLI surface

| Command | Purpose |
|---|---|
| `extenddb init` | Create config, TLS cert, catalog + data databases |
| `extenddb serve` | Start the server |
| `extenddb status` | Check if server is running |
| `extenddb stop` | Stop the server |
| `extenddb destroy` | Drop databases (destructive) |
| `extenddb verify` | Validate config and connectivity |
| `extenddb migrate` | Apply catalog schema migrations |
| `extenddb manage` | IAM administration (users, groups, policies, access keys) |

## Code conventions

- `cargo fmt --all` before committing
- `cargo clippy --all-targets -- -D warnings` with no exceptions
- No `.unwrap()` in library crates; `.expect("reason")` only in `crates/bin`
- Commit messages: imperative mood, 72-char subject line
- DCO sign-off required: `Signed-off-by: Name <email>`
- PR descriptions: What / Why / Testing checklist
